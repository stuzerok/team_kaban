# План верификации регистрового блока UART

## 1. Register map

| Регистр | Адрес | Назначение |
|---|---:|---|
| `RBR_THR` | `0x4000_0000` | RX buffer / TX holding register |
| `DLL` | `0x4000_0004` | Младший байт делителя |
| `DLM` | `0x4000_0008` | Старший байт делителя |
| `LCR` | `0x4000_000C` | Формат кадра и `DLAB` |
| `IER` | `0x4000_0010` | Разрешение прерываний |
| `IIR` | `0x4000_0014` | Идентификатор активного прерывания |
| `FCR` | `0x4000_0018` | Управление FIFO |
| `LSR` | `0x4000_001C` | Статус линии |
| `MCR` | `0x4000_0020` | Управление modem/output |
| `MSR` | `0x4000_0024` | Статус modem lines |

## 2. Ключевые особенности спецификации

### 2.1 `DLAB`-зависимый доступ

Регистры `DLL` и `DLM` доступны только при `LCR[7] = DLAB = 1`.
Это отдельная verification target: нужно проверить как корректный доступ при `DLAB=1`, так и блокировку/альтернативное поведение при `DLAB=0`.

### 2.2 `RBR_THR` с разной семантикой для read/write

Адрес `0x4000_0000` используется по-разному:

- `read` — чтение RX data;
- `write` — запись TX data.

При чтении из пустого буфера должно возвращаться `0x00`.

### 2.3 `RO`-регистры

Регистры `IIR`, `LSR`, `MSR` являются только для чтения.
Нужно проверить, что запись в них не меняет состояние и не вызывает побочных эффектов.

### 2.4 Side effects

- В `FCR` биты `RFRST` и `TFRST` задают сброс FIFO через последовательность `1 -> 0`.
- После чтения `RBR` должны очищаться error flags в `LSR`, кроме случаев, зависящих от реализации.

## 3. Reset verification

После reset проверить:

- адресуемость всех регистров;
- reset value каждого регистра;
- reset value каждого поля;
- reserved bits читаются как `0`, где это определено;
- отсутствие `X/Z` в читаемых битах.

### Ожидаемые reset values

- `RBR_THR[7:0] = 0x00`
- `DLL = 0x00`
- `DLM = 0x00`
- `LCR = 0x01`
- `IER = 0x00`
- `IIR.ID = 0xC`, `IIR.PEND = 1`
- `FCR = 0x00`
- `LSR.THRE = 1`, `LSR.TEMT = 1`, остальные `0`
- `MCR = 0x00`
- `MSR = 0x00`

## 4. Directed test plan

### 4.1 `RBR_THR` @ `0x4000_0000`

Проверки:

- read из пустого RX buffer возвращает `0x00`;
- запись `0x00`, `0xFF`, random byte;
- старшие биты `[31:8]` игнорируются при записи;
- чтение возвращает данные только в `[7:0]`, а `[31:8] == 0`;
- repeated read из пустого буфера стабильно возвращает `0x00`;
- запись в `RBR_THR` не меняет соседние регистры.

### 4.2 `DLL` / `DLM`

Проверки:

- при `DLAB=1` запись/чтение `DLL` и `DLM` работают;
- при `DLAB=0` доступ к ним ведёт себя строго по спецификации/реализации;
- старшие биты `[31:8]` маскируются;
- запись в `DLL` не меняет `DLM` и наоборот;
- комбинация `DLM:DLL` корректно образует 16-битный делитель.

### 4.3 `LCR`

Проверки:

- reset = `0x01`;
- независимая запись/чтение каждого бита: `WLS0`, `WLS1`, `STB`, `PEN`, `EPS`, `STP`, `BRK`, `DLAB`;
- проверка всех комбинаций `WLS[1:0]`;
- reserved bits не влияют на состояние;
- переключение `DLAB` меняет режим доступа к `DLL/DLM`.

### 4.4 `IER`

Проверки:

- reset = `0x00`;
- запись/чтение `RDAIE`, `THREIE`, `RLSIE`, `MDMIE`;
- запись в неописанные биты не меняет состояние;
- отсутствие влияния на другие регистры.

### 4.5 `IIR`

Проверки:

- register read-only;
- reset соответствует `ID=0xC`, `PEND=1`;
- попытка write не меняет значение;
- чтение возвращает только определённые биты;
- проверка `ID` и `PEND` при моделируемых interrupt conditions, если доступны hooks.

### 4.6 `FCR`

Проверки:

- reset = `0x00`;
- запись/чтение `FIFOE`, `RFRST`, `TFRST`, `TFTRG`;
- корректная последовательность `RFRST/TFRST: 1 -> 0`;
- reserved bits не влияют на состояние;
- если `RFRST/TFRST` self-clearing, это должно быть отдельно подтверждено.

### 4.7 `LSR`

Проверки:

- register read-only;
- reset: `THRE=1`, `TEMT=1`, остальные `0`;
- write в `LSR` не меняет состояние;
- чтение стабильно отражает status;
- после чтения `RBR` очищаются error flags, если они были установлены, кроме оговорённых случаев.

### 4.8 `MCR`

Проверки:

- reset = `0x00`;
- запись/чтение `DTR`, `RTS`, `OUT1`, `OUT2`;
- reserved bits игнорируются;
- отсутствие alias с `MSR`.

### 4.9 `MSR`

Проверки:

- register read-only;
- reset = `0x00`;
- write ignore;
- чтение только статусных битов;
- при наличии hooks — проверка изменения edge/status bits.

## 5. Negative testing

Отдельно проверить:

- read/write по адресам вне карты регистров;
- read/write по misaligned address, если master может их генерировать;
- запись в `RO` регистры;
- запись `32'hFFFF_FFFF` в 8-битные регистры;
- чтение после запрещённой записи;
- переключение `DLAB` внутри последовательности доступа;
- back-to-back accesses к одному адресу;
- отсутствие порчи соседних регистров после bad access.
